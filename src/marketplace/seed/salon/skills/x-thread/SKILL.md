# x-thread

## when to use

Use when a user asks for a thread, a tweet, or a multi-post X sequence. Use it too when they ask to turn an idea, launch, or piece of writing into "something for Twitter/X."

## workflow

1. Read `.atelier/memory/voice.md` if present. Match its adjective sliders and vocabulary before drafting a single line.
2. Check `.atelier/memory/instincts.md` for standing overrides (banned topics, posting windows, platform-native rules) and apply them.
3. Draft the arc before the posts: hook, escalation, payoff, close. The hook post earns the click into the thread; the middle posts escalate the stakes or specificity; the payoff delivers the promise the hook made; the close is a command, not a question.
4. Write each post to stand completely alone. A reader who sees only post 4 out of context should still get a complete thought.
5. Count characters as you write, not after. Budget to 280 per post with room to spare — a post that just barely fits is a post that breaks on a client with different link-shortening behavior.
6. Strip any "1/", "2/" numbering. The thread reads as a natural escalation; numbering announces the format instead of letting it work.
7. Pull every external link out of the body and place it in the first reply, once the thread is posted. Treat that first reply as its own piece of content, not a link dump — caption it on purpose, then run three quick self-tests before sending: does it stand alone if someone reads only it? does it add something the thread itself didn't say? would you tap it if a stranger posted it? For a conversion-focused thread, a comment-to-DM funnel ("comment X and I'll send it") usually outperforms a bare link.
8. Reread for open loops: does post 2 make someone want to see post 3? A thread that can be fully absorbed from the hook alone won't earn replies.
9. Post, then stay active in-thread for 30 minutes to answer replies as they land — hand off to a launch-window skill for that window if one is available.

## rules

- 280-char limit per post, counted, not estimated.
- No "1/" or "2/" numbering anywhere in the thread.
- Links go in the first reply only — never in the body of any post in the thread.
- The first reply is a content decision on its own: caption it deliberately, pass the stand-alone / adds-something / would-you-tap-it self-tests, and lean on a comment-to-DM funnel instead of a raw link when the goal is conversion.
- Never edit a live post. If a correction is needed within 30 minutes of posting, delete and repost the whole thread instead.
- Every post must be readable and complete in isolation — no post should require the one before it to make sense.
- Replies and dwell time are the two signals the algorithm weighs most heavily for threads. Write posts that raise a question the reader wants answered, not posts that closed off the topic.
- Profile clicks and bookmarks sit in the next tier down, still worth writing for. A post that makes someone check the profile or save the thread for later is doing real work even without a reply.
- Plan for a 30-minute active window after posting — replies in that window do more for reach than the thread text itself.
