# adversarial-refinement

## when to use

Use when a draft is written but not yet trusted — a high-stakes launch post, a thread the user wants "torn apart before it goes out," or any content where a single self-review pass isn't enough. Pairs well with post-audit: audit checks mechanics, this checks whether the content actually works on a reader.

## workflow

1. Read `.atelier/memory/voice.md` if present. Some persona critiques (especially editor and competitor) will pull toward genericizing a draft — voice.md is the anchor that keeps cuts from flattening the brand's actual personality.
2. Check `.atelier/memory/instincts.md` for overrides on tone or claims before running personas.
3. Pick the depth tier: quick, standard, or deep, based on how much is riding on this piece and how much time is available.
4. Run each persona pass called for by the tier, in order, writing down what each one flags — don't silently fix issues mid-pass, capture them first.
5. After all passes for the tier are done, revise the draft against the collected notes in one pass, rather than rewriting after each persona individually.
6. For a deep pass, set the revised draft aside for 24 hours before the final read — distance surfaces problems immediate re-reading can't.
7. Confirm the closer line is a command, not a question, regardless of tier.

## the five personas

1. **Skeptic**: why should anyone care? If the draft can't answer this in the first few lines, nothing else matters yet.
2. **Expert**: is this actually accurate? Checks claims, numbers, and technical statements for anything that would embarrass the writer in front of someone who knows the subject.
3. **Scroller**: would this actually stop a thumb mid-scroll, or does it read as more of the same? Judges the hook and the first visual impression, not the argument.
4. **Competitor**: could literally anyone else in this space have posted this? Flags anything generic enough to have come from a rival account with the names swapped.
5. **Editor**: cut 20%. Forces a length pass that removes throat-clearing, redundant setup, and any sentence that doesn't earn its place.

## depth tiers

| tier | passes run | revision |
|---|---|---|
| quick | skeptic, scroller, editor (passes 1, 3, 5) | one light revision pass |
| standard | all five personas | one full revision pass |
| deep | all five personas | full revision, then a 24-hour cooling period before final read |

## rules

- Passes run in the stated order; capture each persona's notes before revising, don't fix-as-you-go.
- Quick tier always uses skeptic, scroller, and editor — the three that catch the most common failure (nobody cares, nobody stops, it's too long) fastest.
- Standard and deep tiers run all five personas, no skipping.
- Deep tier requires the 24-hour cooling period before the draft is considered final — don't compress it to save time; that defeats the point of the tier.
- The closer must be a command ("reply with your take," "grab the early-bird link") never a question ("what do you think?") — commands convert better and read as confident.

## checklist

- [ ] Voice file checked and matched (or noted as absent)
- [ ] Instincts checked for overrides
- [ ] Depth tier chosen and stated up front
- [ ] All required persona passes run and notes captured
- [ ] Single consolidated revision pass done (post-cooling-period for deep tier)
- [ ] Closer line is a command, not a question
