# comment-strategy

## when to use

Use when a user asks to comment on someone else's post, build visibility in a niche before they have an audience of their own, or wants a standing routine for showing up in other people's threads. This is not reply-playbook — that skill covers replies to comments on the user's own posts.

## workflow

1. Read `.atelier/memory/voice.md` if present. A comment still carries the account's voice, even at 15-30 words.
2. Check `.atelier/memory/instincts.md` for overrides — banned topics or accounts apply to comments exactly as they apply to posts.
3. Select targets: accounts in the user's niche with a larger following than the user's own, and posts under 30 minutes old. Early comments sit near the top of the thread; a comment on a six-hour-old post is buried before anyone reads it.
4. Pick a comment shape from the menu below based on what the post actually said, not a default habit.
5. Write the comment. Minimum 15 words, and every word has to earn its place — a comment can be short and still be padding.
6. Pace like a person: react to the post, let a beat pass, then comment. A reaction and a comment landing in the same second reads as scripted, and platforms weight it that way too.
7. Run the comment against `rules/social/engagement-ethics.md` before sending. If it's bare agreement, a drive-by plug, or would only work as a sockpuppet, it doesn't go out.

## comment shapes

| shape | what it does | when to use |
|---|---|---|
| missing-piece concession | agree with the post, then name the one thing it left out | default choice — this shape gets the highest measured rate of author replies of any of these seven |
| answer-the-closing-question | the post ends on a question; answer it directly and specifically | any post that closes with an explicit ask |
| data-point-first | open with a number or concrete result, then connect it to the post's claim | posts making a claim you can back or complicate with a real figure |
| practitioner observation | add a detail only someone who's done the work would know | technical or how-to posts in the user's actual domain |
| agree-then-push counter (with concession) | concede the premise first, then argue a sharper or opposite conclusion from it | posts with a take worth pressure-testing, not posts that are factually wrong |
| quotable reframe | restate the post's idea in under 12 words, sharper than the original | strong posts where there's nothing to add, only something to crystallize |
| sharper follow-up question | ask the question the post's own logic raises but doesn't answer | posts that stop one step short of their own implication |

## rules

- 15 words minimum per comment; anything shorter reads as filler even when well-intentioned.
- Target 10-20 meaningful comments per day on niche-relevant posts. Fewer and the channel doesn't compound; more and quality drops before quantity helps.
- Comment within the first 30 minutes of a post going live when possible — that's the window for top-of-thread positioning.
- Target accounts bigger than the user's own. Commenting on smaller accounts doesn't move visibility.
- Sequence is react, pause a beat, then comment — never both in the same instant.
- Never LIKE-react a comment that's arguing a counter-position underneath the post. Liking a counter-argument reads as picking a side against the post's author before you've said a word.
- Every comment passes `rules/social/engagement-ethics.md` before it ships: no bare agreement, no drive-by self-promotion, no sockpuppet-flavored coordination.
