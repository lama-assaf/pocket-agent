# reply-playbook

## when to use

Use when a user has comments sitting on their own published post and needs help deciding what to answer, how to answer it, or how to handle a hostile or off-topic reply. This is not comment-strategy — that skill covers commenting on other people's posts.

## workflow

1. Read `.atelier/memory/voice.md` if present. Replies are the most public-facing writing a brand does; the voice slip that's forgivable in a draft is not forgivable in a reply thread.
2. Check `.atelier/memory/instincts.md` for overrides — standing rules on banned topics or no-response accounts apply to replies too.
3. Triage every comment against the matrix below before drafting anything.
4. For each comment marked amplify, answer, redirect, or defuse, pick a reply shape from the menu and draft it.
5. Apply platform thread mechanics before posting: on LinkedIn, always reply to the top-level comment, not a nested reply — the platform flattens threads past two levels and a buried reply reads as if it never happened. On X, reply to every substantive comment inside the first hour; replies you receive boost the post's own ranking, so hour one is worth the effort.
6. For hostile replies, run the de-escalation script before deciding to answer at all.
7. If the thread is a community raid (a wave of unfamiliar accounts arriving from another community, common around web3 launches), apply raid etiquette instead of the standard matrix.

## triage matrix

| category | criteria | action |
|---|---|---|
| amplify | real audience overlap, a genuine compliment or story that other readers benefit from seeing surfaced | pin or heart, plus a short reply that adds one more detail |
| answer | a real question, in good faith, that other readers likely share | answer directly and specifically |
| redirect | on-topic but better served elsewhere (a DM, a support channel, a pinned thread) | reply pointing to the better channel, don't solve it inline |
| defuse | frustrated or sharp but rooted in a real grievance | de-escalation script, then a substantive answer |
| ignore + hide | spam, bad-faith bait, or a comment designed to provoke rather than discuss | no reply; hide only if it violates platform rules or drowns out real discussion |

## reply shapes

| shape | what it does | when to use |
|---|---|---|
| answer-their-question | direct, specific answer to what was actually asked | answer-category comments |
| concede-then-sharpen | grant the valid part of a pushback, then restate the claim more precisely | defuse-category comments with a real point buried in the tone |
| extend-their-thesis | take the commenter's idea one step further and credit them for it | amplify-category comments from a strong or aligned voice |
| lived-experience | answer with a specific personal or team anecdote instead of a general claim | answer-category comments asking "does this actually work" |
| ask-back | turn the reply into a question that invites the commenter to say more | amplify-category comments worth extending into a conversation |

## de-escalation script for hostile replies

1. Don't reply in the same breath you read it. Let a beat pass.
2. Find the one true thing in the comment, even inside an unfair delivery, and name it first.
3. Correct the factual or scope error once, plainly, without matching the tone.
4. Close with an invitation to continue somewhere calmer (DM, a linked source) rather than a rebuttal that keeps the fight in public.
5. If the reply is bad-faith with no true thing to find, this isn't a defuse case — move it to ignore + hide.

## community-raid etiquette (web3)

- Welcome raiders with substance: answer their actual questions, don't post a generic "welcome fam" line.
- Redirect the energy: point new arrivals to a pinned thread or FAQ rather than letting the raid scatter across every reply.
- Don't treat raid volume as validation to report upward — a raid is attention, not qualified interest, until individual accounts show up again later.

## rules

- LinkedIn threads flatten to two levels — always reply to the top-level comment.
- On X, reply to every substantive comment within the first hour of the post going live; replies you receive are a ranking signal for the post.
- Hostile replies go through the de-escalation script before any answer is drafted, not after.
- Bad-faith or spam comments get ignore + hide, not a reply that tries to win them over.
- Raid welcomes point to substance and a pinned thread, never a copy-pasted greeting.
- Every reply passes `rules/social/engagement-ethics.md` — no paid disclosure omitted, no coordinated pile-on framed as organic.
