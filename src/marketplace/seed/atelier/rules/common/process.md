# process

how atelier work flows.

## start with the why, not the what

before designing a screen, writing a prd, or naming a product, write down the situation that triggers the need and the user it affects. if you can't, you're not ready to make the artifact.

## one artifact, one purpose

a prd is not a marketing plan. a positioning statement is not a roadmap. when one artifact tries to do two jobs, both jobs suffer.

## drafts are private. shared work is reviewed.

drafts are messy on purpose. shared work has been read aloud, checked against the brief, and revised at least once.

## review in writing first, meeting second

written review forces precision. meetings are for resolving disagreement, not for hearing the work for the first time.

## sleep on big decisions

design choices, hires, and public commitments benefit from a second look 24 hours later. the version you ship is rarely the first version that felt right.
