# handoff

how work moves between people.

## handoffs are documents, not channels

a handoff is a written artifact. a slack message linking to a figma file with no context is not a handoff.

## include the why

the receiver of a handoff needs the why, not just the what. "build this button" is incomplete; "build this button because users in our research said the affordance for action X was unclear" gives the receiver the basis to make the right tradeoffs.

## states before edges

list every state the system can be in before listing edge cases. states are first-class; edges are exceptions to states.

## include the open questions

handoffs that hide open questions force the receiver to discover them. handoffs that name them save round trips.

## acceptance criteria before timeline

when does the receiver know it's done? write that down. the answer is rarely "ship date".
