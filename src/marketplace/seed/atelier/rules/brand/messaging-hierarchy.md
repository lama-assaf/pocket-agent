# messaging hierarchy

every brand has a structure:

```
hero message (one line)
└── pillar 1 (claim)
    ├── proof point
    ├── proof point
    └── proof point
└── pillar 2 (claim)
    └── ...
└── pillar 3 (claim)
    └── ...
```

## rules

- 3 pillars is the sweet spot. 4 maximum.
- each pillar is a claim, not a feature.
- pillars don't overlap.
- every pillar has 3-5 specific proof points.

## audience variants

if you serve multiple audiences, the same hero and pillars may shift in framing:

- different language for technical vs business audiences
- different proof points emphasized
- different sub-pillars at the bottom

don't multiply variants beyond what you can keep consistent.

## what to avoid

- pillars that are features dressed up as claims
- pillars without supporting proof
- different teams (sales, marketing, product) using different language for the same things
- messaging that hasn't been revisited in a year
