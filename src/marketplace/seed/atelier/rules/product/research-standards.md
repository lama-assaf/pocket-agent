# research standards

how research is done at quality.

## read everything before synthesizing

never synthesize from a sample if the full set is available.

## tag, don't paraphrase

keep original wording. tag for job, frustration, workaround, desire, hard constraint. paraphrasing loses signal.

## cluster by need, not by surface language

users describe the same job in different words. cluster by underlying need.

## count

per cluster: how many participants, how often, how strongly.

## name themes in user language

themes get names that come from participants, not from the team.

## what to avoid

- invented quotes
- generalizing from one passionate participant
- recommending solutions before themes stabilize
- presenting themes in decks with conclusion before evidence
- softening adverbs ("most", "many", "some") without numbers behind them
