# metrics

four metric types per initiative:

- **north star**: long-term, captures the value the product creates
- **primary**: one per initiative, moveable by the work
- **secondary**: 1-3, watched but not optimized
- **guardrails**: things that must not break

## the test

before adopting a metric:

- can the team move it with their work
- is it observable in a useful timeframe
- is it gameable in ways that hurt the user
- does it capture the real thing or only a proxy

## leading vs lagging

pair one of each. leading shifts before the thing you care about. lagging is the truer signal but slower.

## anti-patterns

- vanity metrics (page views, signups)
- "improve" as a target
- metrics with no defined query
- changing the metric mid-initiative because the trend is bad
- tracking everything
