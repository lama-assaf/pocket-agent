# prd structure

a prd captures: the problem, the goal, the scope, the approach, the risks.

## sections

1. context (one to two sentences)
2. problem (who, what, evidence, current workaround)
3. goal (jtbd framing, primary metric, secondary metrics, guardrails)
4. scope (in, out, alternatives considered)
5. approach (user flow, key screens, data shape, states)
6. open questions
7. risks (what could kill this, what we're betting on, what to learn first)

length is the wrong metric. clarity is.

## rules

- write in present tense
- name a real user, not a hollow persona
- use numbers where you have them; name the gap when you don't
- one prd, one feature

## the test

a prd is ready to share when a reader who wasn't in the conversation can:

- restate the problem
- name the success metric
- describe the scope
- identify the open questions

if any of those fail, the prd isn't ready yet.
