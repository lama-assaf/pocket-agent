# jobs to be done

framing user needs by job:

```
when [situation],
i want to [motivation],
so i can [outcome].
```

## the rules

- situation is a moment, not a role
- motivation is the user's verb, not the product's
- outcome is the success state the user can measure

## three layers

every job has functional, emotional, and social layers. great products address all three.

## how to find the right altitude

ask why five times. the surface request is usually not the job.

## anti-patterns

- "as a user, i want a feature, so i can use it" — not a job
- jobs that are really roles ("when i'm a marketer")
- jobs that are really features ("when i need to filter")
- jobs only one user has
