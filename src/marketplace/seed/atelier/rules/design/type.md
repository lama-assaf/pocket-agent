# type

## scale

a type ramp has 5-8 sizes. typical scale:

```
display: 48-64px (rare, hero moments only)
h1: 32-40px
h2: 24-28px
h3: 20px
body: 16px
small: 14px
caption: 12px
```

scale ratio is consistent. major third (1.25x) is the default; minor third (1.2x) for tighter scales; perfect fourth (1.33x) for looser.

## weights

3 weights at most: regular, medium (or semibold), bold.

avoid mixing too many weights in one block. headlines may be one weight; body text is another. captions usually share the body weight.

## line-height

- body text: 1.5x
- ui labels and short lines: 1.2-1.3x
- headlines: 1.1-1.2x

line-height tightens as size grows. 16px body needs 24px line-height. 40px headline needs 44-48px.

## tracking (letter-spacing)

- body: default (no tracking adjustment)
- headlines: -1% to -2% (slightly tightened)
- small caps and uppercase labels: +5% to +10% (slightly loosened)

## measure (line length)

body text reads best at 50-75 characters per line. wider lines fatigue. narrower lines feel choppy.

## fonts

- one display face for headlines (or shared with body)
- one body face
- one mono if needed (code, numerical data)

if using a single typeface for everything, vary by size and weight to create hierarchy.

## what to avoid

- mixing 14 and 15 (or any two adjacent sizes) in the same context
- using more than 3 weights
- centered body text (left-aligned reads faster)
- justified text in narrow columns (creates rivers)
- font-size below 14px for sustained reading
