# /value-prop

write a value prop that connects what a product does to what a user gets.

## what it does

builds:

```
hero (one line): for [audience], [product] is [category] that [outcome]
support (one paragraph): the unpacked version
proof (bullets): concrete evidence
variants (per audience): aligned framings
what we don't say: claims we could make but won't
```

## rules it follows

- specific audience (one ear)
- concrete outcome (user-facing result, not feature)
- honest differentiation (what's really different)
- user language (words your audience uses)

## how to use

```
/value-prop
```

then provide audience, what they're trying to do, what they try today, what bothers them.
