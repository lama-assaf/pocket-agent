# /messaging

build the structure that makes copy consistent across pages, decks, ads, and conversations.

## what it does

builds:

```
hero (one line)
└── pillar 1 (claim)
    ├── proof point
    ├── proof point
    └── proof point
└── pillar 2
└── pillar 3

audience variants
channel variants
what we don't say
```

## rules it follows

- 3 pillars is the sweet spot, 4 maximum
- each pillar is a claim, not a feature
- pillars don't overlap
- every pillar has 3-5 specific proof points
- audience variants share the hero and pillars but adjust framing

## how to use

```
/messaging
```

then provide product, primary audience, secondary audiences, what you want them to believe.
