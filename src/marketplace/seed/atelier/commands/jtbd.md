# /jtbd

frame a product, feature, or user need using JTBD methodology.

## what it does

produces:

```
job: when [situation], i want to [motivation], so i can [outcome]
who hires: [user type and situation]
who they fire (alternative): [current option]
emotional layer: [how they want to feel]
social layer: [how they want to be seen]
evidence: [where the job came from]
```

## how to use

```
/jtbd
```

then describe the user, the surface request, and what you know about their situation. the command will ask "why?" a few times to find the right altitude.

## rules it follows

- situation is a moment, not a role
- motivation is the user's verb, not the product's
- outcome is the success state, observable
