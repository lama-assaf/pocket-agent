## 2026-01-15 — feature shipped without metric

context: shipped a feature for power users, didn't define a success metric upfront. four weeks later, no one could tell if it worked.

lesson: a metric (even a rough one) goes in the prd before any code is written. if no metric, the feature isn't ready to scope.

cost: a quarter of build time on something we can't evaluate.

---

## 2026-02-03 — brand voice extraction skipped, copy drift accelerated

context: started writing landing copy and email sequences without an extracted voice guide. three writers contributed; three voices showed up.

lesson: voice extraction is a one-day investment that saves weeks of revision. do it once, then never let new copy ship without checking against the guide.

cost: 2 weeks of revisions, plus the perception that the brand is inconsistent.
